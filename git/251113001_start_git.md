## Git? GitHub?
Git은 시스템, GitHub는 서비스로 서로 다른 기능
## Git 개념정리
- Commit : 시간 단위 (예: 작업 시간 기준 커밋)
- Branch : 공간 단위 (예: 기능, 작업 영역 별 분리)

## Git Flow
1. clone 으로 레포지토리 연결
2. .gitignore 구성
3. 변경 사항 발생시add, commit
4. 일정 작업이 완료되면 push

## Commit
- feat : 기능 개발 관련
- fix : 오류 개선/버그 패치
- docs : 문서화 작업
- test : test 관련
- conf : 환경 설정
- build : 빌드 작업 관련
- ci : Continuous Intergration 관련
- chore : 패키지 매니저, 스크립트 등
- style : 코드 포매팅 관련
- refactor : 코드 리팩토링

## .gitignore
* 특정 파일 혹은 디렉토리 추적 방지
** OS, Framework, Editor, 썸네일, 캐시 파일 등에 따라 확인
** [reference:](gitignore.io) 사이트 참조

## pre-commit
* commit 전 체크사항을 자동 수행하도록 도와주는 도구
** [reference:](https://pre-commit.com/)

## Branch
보통 Branch 이름은 작업 내용으로 명시

* git branch : local branch 조회
** -r : remote branch 조회(`*` 문자로 현재 영역 구분)
** -a : 모든 branch 조회
** -D {branch}: 삭제(작업 종료 후 항상 삭제하는 습관 들이기)

* git branch {branch} : 생성
* git switch {branch} : branch 변경
* git merge {branch} : branch 병합 (합치고자 하는 branch에서 실행)
** 분기된 지점으로 돌아가는 것이 가장 안전함
